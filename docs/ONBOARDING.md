# Onboarding

## Goal

LongTable onboarding은 기술 설정이 아니라 epistemic onboarding이어야 한다.

## Principle

사용자를 시스템에 동화시키는 것이 아니라, 시스템이 사용자의 연구 스타일과 governance 선호를 학습하는 출발점이어야 한다.

## Two-Stage Onboarding

### Stage 1: Quick Setup

Target time: 2-3 minutes

Collect:

- career stage
- current project type
- current priority: novelty / balance / defensibility
- preferred checkpoint intensity
- weakest domain: theory / design / measurement / analysis / writing

### Stage 2: Progressive Calibration

Collect gradually through use:

- ambiguity tolerance
- AI autonomy tolerance
- explanation depth preference
- frequent tensions
- preferred interaction mode defaults

## Output

Onboarding produces:

- initial researcher profile
- initial checkpoint policy
- initial memory seed

## Non-Goal

Do not ask 20 setup questions up front.
