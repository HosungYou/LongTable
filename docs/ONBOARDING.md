# Onboarding

## Goal

LongTable onboarding은 기술 설정이 아니라 epistemic onboarding이어야 한다.

## Principle

사용자를 시스템에 동화시키는 것이 아니라, 시스템이 사용자의 연구 스타일과 governance 선호를 학습하는 출발점이어야 한다.

## Two-Stage Onboarding

### Stage 1: Runtime Permission Setup

Target time: 2-3 minutes

Collect:

- provider
- install scope
- approved runtime surfaces
- intervention intensity
- provider-native interview launch preference

### Stage 2: Provider-Native Project Interview

Collect when `$longtable-start` creates or resumes a workspace:

- rough research interest
- still-hard-to-say problem
- concrete scene, case, material, text, dataset, or decision
- answer-quality signals
- provisional First Research Shape
- final structured confirmation

### Stage 3: Progressive Calibration

Collect gradually through use:

- ambiguity tolerance
- AI autonomy tolerance
- explanation depth preference
- field or disciplinary context
- frequent tensions
- preferred interaction mode defaults

## Output

Onboarding produces:

- initial researcher profile
- initial checkpoint policy
- initial memory seed

## Non-Goal

Do not ask 20 setup questions up front.
